# CX Knowledgebase : CX SIP Proxy as a CUSP replacement

CX SIP Proxy can be used as a replacement of Cisco Unified SIP Proxy (CUSP). See the following table for a feature comparison:

**Feature**| **Cisco CUSP**| **CX SIP Proxy**  
---|---|---  
Proxy for SIP unified communications signaling| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
Signaling support| ![\(tick\)](images/icons/emoticons/check.png) (voice)| ![\(tick\)](images/icons/emoticons/check.png) (voice)  
Address resolution| ![\(tick\)](images/icons/emoticons/check.png) (DNS: Type A and SRV and Type NAPTR)| ![\(tick\)](images/icons/emoticons/check.png) (DNS: Type A and SRV, ENUM)  
TCP, User Datagram Protocol (UDP), and Transport Layer Security (TLS)| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
Redundancy and high availability options| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
SIP header manipulation| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
Support for IPv4 and IPv6| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
Call admission control| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
Dynamic call routing and load balancing| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
QoS support| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
Monitoring and reporting through syslog, SNMP, and SNMP traps| ![\(tick\)](images/icons/emoticons/check.png)| ![\(tick\)](images/icons/emoticons/check.png)  
  
The scenarios for replacing CUSP with CX SIP Proxy for outbound and inbound calls are given [here](Call-Scenarios-for-CUSP-Replacement_687178260.html).
