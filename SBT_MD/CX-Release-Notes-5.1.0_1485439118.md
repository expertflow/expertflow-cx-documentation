## New in CX5.1.0

This is a patch release on top of CX5.0, which includes targeted enhancements and critical fixes to improve security and overall platform stability.

### Platform & Security Improvements

**Feature(s)**| **Description**  
---|---  
**CX Bulk Customer Upload**|  Improves CX Bulk Customer Upload to automatically detect existing customers using channel identifiers, prevent duplicates, and intelligently merge or update records based on admin‑selected options. The upload summary now clearly shows how many records are inserted, ignored, updated, replaced, appended, or rejected. This reduces manual clean‑up, improves campaign targeting accuracy, and lowers the risk of over‑contacting the same customer.  
**Improve Stability for GraalVM-Based CX Deployments**[**(shipped in 4.10.2)**](https://expertflow-docs.atlassian.net/wiki/spaces/CX/pages/1297416214/4.10+Patch+Release+CX4.10.2)**-** to be discussed| Improves the stability of CX deployments running on GraalVM by enhancing how security certificates are handled. Environments using GraalVM now manage certificates more reliably during builds and runtime.  
**Security Enhancements and Vulnerability Hardening**|  Strengthens the security posture of CX 5.x by addressing vulnerabilities previously identified in the CX 5.0 components. These hardening measures remove known critical and high‑risk exposure points and align the platform with current security best practices. Customers are encouraged to upgrade to CX 5.1.0 to benefit from the improved security baseline.  
**Improve Platform Security Based on Penetration Assessment**|  Improves the CX platform security based on findings from the CX 5.0 penetration assessment. This hardens multiple components to reduce the attack surface and increase protection against potential threats. These improvements target the underlying infrastructure and configuration rather than user‑visible behavior, so administrators upgrade to this release and get enhanced security without changing existing workflows or settings.  
**Security Enhancement and Vulnerability Hardening**|  Strengthens the security posture of CX 5.x by addressing high and critical vulnerabilities previously identified in core services and supporting components. We updated and rebuilt container images and dependencies so that all affected services pass Trivy v2 security scans without any known critical or high‑severity issues, except for documented false positives or vulnerabilities with no upstream fix yet available.

>   * Customers can request a detailed vulnerability assessment report from Expertflow Support.
>   * For detailed findings, see the updated Vulnerability Report for CX Components 5.1.x.
>   * After these improvements, CX 5.1.x has no known open critical or high‑severity vulnerabilities.
> 
  
  
[**Vault‑Backed Service‑Based Database Accounts**](Vault-Configuration_1624473620.html)|  Each CX service now uses its own least‑privilege database credentials managed in Vault, improving tenant and service isolation. This reduces configuration drift and connectivity issues during deployments and upgrades, with no changes to agent or supervisor workflows.  
  
[**Conversation Search**](https://expertflow-docs.atlassian.net/wiki/spaces/CIMT/pages/1099792618/Conversation+Search+API+using+GraphQL+-+User+Guide)**and GraphQL APIs**|  The `searchConversation` [GraphQL API ](https://api.expertflow.com/#c94f9abf-5f0a-4667-8d52-4805b9fd7808)now returns channel information for each conversation, so clients can immediately identify whether a conversation came from web, email, WhatsApp, or another channel. The API also supports a generic `searchTerm` parameter that lets clients run free-text searches, including by customer `firstName`, without specifying a particular field. These enhancements improve how UIs and integrations search and display conversation history.  
**Standardize Fluent Bit logging via the built‑in Helm chart**|  Fluent Bit is now part of the standard CX platform Helm chart. The platform enables centralized logging for all Kubernetes workloads by default, reducing manual setup. For configuration details and advanced usage, see the [this guide](Deploy-Fluentbit-for-Opensearch_1278410772.html).  
[**Partial updates for Form Schema**](https://docs.expertflow.com/cx-knowledgebase/latest/form-builder-revamp#FormBuilder-PartialUpdatesforFormSchema\(HTTPPATCH\))|  Introduces a targeted and reliable way to update forms. Rather than resending the entire form for a small correction, the system now transmits only the specific change, such as a corrected typo, minimizing unnecessary updates.  
****[**Password Management for EFCX Users**](/wiki/pages/createpage.action?spaceKey=SBT&title=IAM%20%28Keycloak%29%20Password%20Policies%20-%20User%20Guide&linkCreation=true&fromPageId=1485439118)|  The system now forces all EFCX users to change temporary passwords on their first login to AgentDesk and Unified Admin. Users can change their passwords anytime from their profile, and the system logs them out and requires re-authentication with the new password to maintain account security.  
[**Improved Traceability and Compliance in Unified Admin**](Audit-Logging_1578795038.html)|  Unified Admin now records audit logs for key configuration changes, including teams, pull mode lists, reasons, agent desk settings, and locale settings. The system formats logs using our [standard logging guide](https://expertflow-docs.atlassian.net/wiki/spaces/CP/pages/750911551/Logging+and+Tracing+Architechture), follows OpenTelemetry (OTLP), and forwards them to Kube Logging so you can search and analyze them in OpenSearch Dashboards.  
**Improved tenant scalability**|  Creates more than 10 tenants without impacting existing tenants. The platform removes the previous limitation that made the first tenant unavailable after adding the 11th tenant, so multi-tenant deployments scale more reliably.  
**Enable multi‑tenant authentication via CX Application Gateway**|  Enhances CX API Gateway authentication to validate tokens against tenant-specific realms in multi-tenant deployments. The gateway now selects the correct realm at runtime based on tenant context or token metadata, while existing single-tenant deployments continue to work without any configuration changes.  
**Display password policy guidance when updating user passwords**|  The UI now shows password policy requirements when users change their passwords. Users can view policy details via an info icon and see which rules their new password meets or fails, using clear visual indicators. This helps users create compliant passwords on the first attempt and reduces failed password updates.  
  
### Tenant, Branding & Multitenancy

**Feature(s)**| **Description**  
---|---  
**Tenant-specific logo management for CX Applications**|  Tenant admins can now upload, update, and remove tenant-specific logos from the tenant configuration so CX applications (Unified Admin and Unified Agent) display the organization’s logo instead of the default ExpertFlow branding. The applications load the logo dynamically from the File Engine on login screens and in-app headers, and fall back to the default logo if no tenant logo is available. For consistent display across all placements, use a logo with a **4:1 aspect ratio**.  
**Support multiple media servers per tenant**|  Allows each tenant connect to its own media server by configuring media server URLs through the tenant service API, which automatically creates the corresponding domain and stores the settings centrally. Agent Desk now reads media server details from the CX Tenant service instead of static configuration variables, improving flexibility and isolation for multi-tenant deployments.  
  
#### Voice and Customer Experience

**Feature(s)**| **Description**  
---|---  
**Manual Outbound Call Tracking for CX Voice**|  Creates CX conversations as soon as agents press **Dial**. The system consistently captures every outbound attempt, successful or failed, and logs standardized failure reasons and detailed timing metrics (start time, end time, duration, talk time, and hold time) to fully reflect manual outbound activity in historical, summary, and conversation-level reports.  
  
### Reporting & Data Platform

**Feature(s)**| **Description**  
---|---  
**Standardize Campaign Gold Tables in the Data Platform**|  Campaign reporting gold tables are now managed in the Data Build Tool (DBT), but table names and schemas stay the same, to keep existing reports and integrations working.

  * For environments **using the Data Platform** , follow the [guide](https://expertflow-docs.atlassian.net/wiki/spaces/CXDOC/pages/1695219745/5.1+Upgrade+Guide+for+Data+Platform+-+Draft) for the required upgrade steps.
  * For environments with **existing campaign history** , follow the Data Platform upgrade steps in the [guide](Upgrade-Guide-CX5.0.2-to-CX5.1.0_1486520327.html) to run the schema migrations.
