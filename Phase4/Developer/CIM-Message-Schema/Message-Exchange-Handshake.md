# Message Exchange Handshake

To ensure reliable delivery and real-time responsiveness, ExpertFlow CX utilizes an event-driven handshake for all message exchanges. This process ensures that every actor (Agent Desk, Bot, Connector) is synchronized with the Platform's state.

## 1. The Exchange Components
Every exchange involves three distinct segments:
1.  **Unique Identifier (ID):** Generated by the sender to track the message lifecycle.
2.  **Metadata (Header):** Informs the system of the routing context.
3.  **Content (Body):** The payload being delivered.

## 2. Handshake Lifecycle (WebSocket)
When an agent sends a message via the Agent Desk:
1.  **Emit:** The Agent Desk emits the `publishCimEvent` Socket.io event.
2.  **Acknowledgment:** The Platform returns a system acknowledgment to confirm the message was received.
3.  **Broadcast:** The Platform emits `onCimEvent` to all other participants in the conversation (including other agents or supervisors).

## 3. Data Integrity
- **Persistence:** Messages are only stored in the **Reporting Database** after a successful handshake. 
- **Retry Logic:** If an actor fails to receive an acknowledgment within the preconfigured timeout, it should initiate a retry with the same `originalMessageId` to prevent duplication.
