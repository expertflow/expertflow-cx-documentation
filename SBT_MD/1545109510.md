# CX Knowledgebase : IAM (Keycloak) Configuration 4.10.X

This document covers the setup for importing the pre-configured realm in Keycloak.

  1. For release HC 4.1 or backward download the [realm-export.json](https://expertflow-docs.atlassian.net/wiki/download/attachments/2530152/realm-export.json?version=1&modificationDate=1684410375163&cacheVersion=1&api=v2) file and place it in any directory of your choice.

  2. For release CX-4.1 and onwards download the [realm-export-4.1-onwards.json](https://expertflow-docs.atlassian.net/wiki/download/attachments/2530152/realm-export-4.1-onwards.json?version=2&modificationDate=1688468764319&cacheVersion=1&api=v2) file and place it in any directory of your choice.

  3. For release CX-4.3 and onwards download the [realm-export-4.3.json](https://expertflow-docs.atlassian.net/wiki/download/attachments/2530152/realm-export-4.3.json?version=2&modificationDate=1698039528632&cacheVersion=1&api=v2)[ ](https://expertflow-docs.atlassian.net/wiki/download/attachments/2530152/realm-export%20-4.3.json?version=1&modificationDate=1697813227265&cacheVersion=1&api=v2)file and place it in any directory of your choice.

  4. For release CX-4.5.1 download the [![](attachments/thumbnails/1545109510/1545109534)](attachments/1545109510/1545109534.json) file and place it in any directory of your choice.

  5. For release CX-4.5.2 and onwards download the [![](attachments/thumbnails/1545109510/1545109528)](attachments/1545109510/1545109528.json) file and place it in any directory of your choice.

  6. For release CX-4.7 and onwards, download the [![](attachments/thumbnails/1545109510/1545109525)](attachments/1545109510/1545109525.json) file and place it in any directory of your choice. This file doesn't contain permissions for Campaigns and Surveys, they should be imported separately.

  7. For release CX-4.10, download the [![](attachments/thumbnails/1545109510/1545109522)](attachments/1545109510/1545109522.json) file and place it in any directory of your choice. This file doesn't contain permissions for Campaigns and Surveys, they should be imported separately.

  8. For release CX-4.10.1, download the [![](attachments/thumbnails/1545109510/1545109516)](attachments/1545109510/1545109516.json) file and place it in any directory of your choice. This file doesn't contain permissions for Campaigns and Surveys, they should be imported separately. API Authorization became available from this release onwards.

  9. If you've not imported the realm and its users using the above realm file & upgrade script, then you can create a realm, client, roles, and an admin user following these steps:   
  
Login to the KeyCloak Administration console by browsing your Keycloak server URL: https://<server-fqdn>/auth.   
  
![](attachments/1545109510/1545109690.png?width=584)

  10. Log in to the administration console using the password set in environment variables (KEYCLOAK_USERNAME and KEYCLOAK_PASSWORD). Default username is **admin** and password is **admin.**

  11. Bring the cursor to the master realm and click on **Add realm** button.  


![](attachments/1545109510/1545109603.png?width=578)
  12. Click on the **Select file** button and select the realm-export.json file downloaded in step 1 and press **Create.**  


![](attachments/1545109510/1545109603.png?width=578)
  13. This will create the expertflow realm with all required configurations.  


![](attachments/1545109510/1545109600.png?width=578)

  14. Once realm is imported, we need to create the users in our example **agent/supervisor** and **admin**. To create a user click on **Users** from the left nav bar.  


![](attachments/1545109510/1545109597.png?width=578)
  15. Once the user is created, we can see additional fields at the top (Details, Attributes, Credentials). Now select the Credentials tab and click on **Set Password** button**.** Type the password of your choice, also turn **Temporary** to **off**  
  
![](attachments/1545109510/1545109594.png?width=584)  


  16. To assign the roles to the user, click on the Role Mappings tab on top and click on **Assign Role** Button. In the pop-up window, we can see **admin** , **agent and supervisor** roles, for agent user assign **agent** role and for supervisor user assign **supervisor** role. Assign **offline_access** and**uma_authorization** roles as well.

**agent** role must be assigned to the Agent user if **agents_permission** group is assigned. **supervisor** role must be assigned to the Supervisor user if **agents_permission** and **senior_agents_permission** groups are assigned.

  
**4.5.1 Onwards:** Supervisor and Senior Agent must be assigned **senior_agents_permission** group. **senior_agents_permission** became superset of **agents_permission** group (If we assign senior_agents_permission then all the permissions of agents_permission will be automatically assigned to agent/supervisor).

![](attachments/1545109510/1545109591.png?width=578)
  17. For admin-user first select the **admin, offline_access and uma_authorization** from Roles and click on **Assign** button. This will assign realm-roles to the admin user in Keycloak.  
  
![](attachments/1545109510/1545109585.png?width=584)  


  18. Select the admin user from users list and click on **Assign Role** Button.  
  
![](attachments/1545109510/1545109582.png?width=584)  


  19. Select the **Filter by clients** filter and search all **realm-management** roles**.** Select all realm-management roles and click on **Assign** button.

Assign all 23 roles to **admin** user i.e 19 **realm-management** roles, **admin** role, **default-roles- <Realm-Name>** role, **offline_access** and **uma_authorization** roles.

![](attachments/1545109510/1545109579.png?width=578)
  20. A user shouldn’t have more than one custom CX roles assigned to it.

     * If user has an admin role then it shouldn’t have an agent/supervisor or any other role assigned to it. Same for all other roles, one users should only have one CX role assigned to it apart from uma_authorization, offline_access and default-roles-expertflow.

  21. Follow [User's Teams Configuration](Unified-Admin-Guide_2524407.html) guide to create and assign users to the team.

  22. Team assignment is mandatory for **agent** and **supervisor** roles only. Users with other roles i.e (i.e **admin, routing-manager, quality-manager** etc) are not required to be assigned to a team. Make sure only users with agent or supervisor roles are part of team. Remove all user from teams with neither of role assigned to them, its mandatory.
