# CX Knowledgebase : Twitter Connector – Limitations & Key Highlights

### Limitations

  1. **Bot Response Limitation**

     * Twitter connector **does not support bot responses**.

     * Bot replies may appear as DMs in Agent Desk but will **not be forwarded** to Twitter as public comments or social posts.

  2. **Direct Messaging (DM) Not Supported**

     * Twitter DMs are **not supported**.

     * If a message is sent without a “comment reply action button,” it will display in Agent Desk but **will not be delivered** to Twitter.

  3. **Media Display in View Full Post**

     * Posts containing media **do not display the media** in “View Full Post.”

     * Instead, a link to the original Twitter post page is provided.

     * **Media in comments** , however, will display properly.

  4. **Multiple Media Attachments in Comments**

     * While Twitter **supports multiple media files** in a single comment,

     * Expertflow CX **only displays one** media file per comment.

     * This limitation is due to the CIMMessage structure which supports only one attachment.




* * *

### Important Information & Constraints

  * **Image Message**

    * Max size: **5 MB**

  * **GIF Message**

    * Max size: **15 MB**

  * **Video Message**

    * Twitter supports up to **512 MB**

    * Expertflow CX limits this to **25 MB** for video uploads

  * **Message Forwarding Behavior**

    * Non-comment responses without action buttons will be shown in the agent interface but **not published** on Twitter.

  * **Supported Channel Mode**

    * Twitter connector operates in **PULL MODE** , fetching messages periodically within usage limits.
