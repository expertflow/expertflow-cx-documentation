# CX Knowledgebase : Forms and Form Data

## Form

**Description**  
---  
A form is a skeleton to store structured FormData.  
  
## FormData

FormData is stored as a type of structured message. FormData can be stored as part of an activity, conversation, or channel data. It is contextual to the action of the actor whether they push this as an activity, channel data, or conversation data. The context is provided in the table below:

**Type**| **Description**  
---|---  
Conversation Data| Conversation Data is a set of dynamic key-value pairs that any agent or 3rd party application may use to get, set, or update some information for an active Conversation object. For details, see possible use cases of [conversation data](ConversationData_2526208.html).  
Channel Data| The data of the channel type opted by the customer for conversing with the agent. Channel data is unique for every channel. For details, see [channel data](https://expertflow-docs.atlassian.net/wiki/spaces/CX/pages/6259189/4.3+channelData) object.  
Activities| See [CX Activities](Messages%2C-Events%2C-and-Activities_2528021.html) for details.
