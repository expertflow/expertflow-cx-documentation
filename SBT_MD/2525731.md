# CX Knowledgebase : Reports Configuration (Superset 2.0)

This document covers the setup for importing the reports in the Apache Superset 2.0 Instance.

**1.** Download **for MSSQL →**[![](attachments/thumbnails/2525731/2575125)](attachments/2525731/2575125.zip) and [![](attachments/thumbnails/2525731/1165721773)](attachments/2525731/1165721773.zip) **for MySQL →**[![](attachments/thumbnails/2525731/2575129)](attachments/2525731/2575129.zip) , [![](attachments/thumbnails/2525731/1165787287)](attachments/2525731/1165787287.zip) , [![](attachments/thumbnails/2525731/1030094988)](attachments/2525731/1030094988.zip) and [![](attachments/thumbnails/2525731/1165721766)](attachments/2525731/1165721766.zip) files. Place it in any directory of your choice.

Don't import **both** MsSQL & MySQL reports on the same Apache superset server.

**2.** Unzip this file and go to the directory **databases.**

![](attachments/2525731/2575169.png?width=680)

**3.** Now open file "cim_reporting_database.yaml".

![](attachments/2525731/2575165.png?width=680)

**4.** The field **sqlalchemy_uri** is the connection string in the below format. Please update this as per your configuration. (Don't update any other field.)

**MySQL Format :**[mysql+](https://docs.expertflow.com/mssql+pymssql:/Username:Password@ServerAddress:PortNumber\(Default:1433\)/Databasename)mysqldb://Username:Password@ServerAddress:PortNumber/DatabaseName  
**MsSQL Format :**[mssql+](https://docs.expertflow.com/mssql+pymssql:/Username:Password@ServerAddress:PortNumber\(Default:1433\)/Databasename)pymssql://Username:Password@ServerAddress:PortNumber/DatabaseName

**5.** Now delete the previous zip file & zip the parent folder again.

![](attachments/2525731/2575161.png?width=680)

**6.** Log in to the Superset Administration console by browsing  _https://server-ip:port/_ URL.

![](attachments/2525731/2575173.png?height=250) ****

**7.** Goto the "Import dashboards".

![](attachments/2525731/2575157.png?width=680)

**8.** Select the zip file and press "IMPORT".

![](attachments/2525731/2575153.png?width=680)

**9.** Everything is up now. 

![](attachments/2525731/2575149.png?width=680)

**10.** Now set the UTC offset for reports by the following document [UTC Offset - Reports](UTC-Offset---Reports_2526326.html).

**11.** To delete the previous report to import the new one. Follow this document [here.](https://expertflow-docs.atlassian.net/wiki/x/gwDzAg)

## Superset SSL Configuration

  1. Open the `myCert.cert` file in any text editor and copy its value as shown in the below screenshot.  
  
![](attachments/2525731/42500609.png?height=250)

  2. Login into Superset as admin then go to the**Database Connection** from the **Settings** and edit the Database Connection next go to the **Advance Tab** and paste the copied MySQL certificate text into the **Root Certificate** and click on Finish.  
  
![](attachments/2525731/42500614.png)
