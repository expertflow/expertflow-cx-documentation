# Handling Customer Interactions

As an Agent (Amy), your main screen (Agent Desk) is where all incoming customer requests arrive. This guide explains how to accept and interact with these conversations efficiently.

## 1. Receiving a New Request
When a customer reaches out via any digital channel (WhatsApp, Webchat, Facebook), a notification appears on your desk.

### The Incoming Notification:
- **Accept Button:** Click to join the conversation.
- **Customer Name:** If the system recognizes the customer (e.g., matching a WhatsApp number), the name will appear in **Green**.
- **Anonymous (Jane Doe):** If no match is found, the notification will appear in **Red** with a generic name.

## 2. Managing the Conversation View
Once you accept, the central view expands to show:
- **Conversation History:** All past messages, voice calls, and wrap-up notes for the customer.
- **Customer Info (Right Sidebar):** Displays the customer's profile details.
- **Active Sessions:** Shows all channels the customer currently has open.

## 3. Customer Profile Matching (Lookup)
ExpertFlow CX automatically matches customers based on their **Channel Identifiers** (e.g., Phone Number, Email).

### Common Scenarios for Amy:
1. **Single Match Found:** You can start responding immediately. 
2. **Multiple Matches:** Use the **Link Profile** tab in the sidebar to select the correct customer record. 
3. **No Match Found:** Update the anonymous "Jane Doe" profile with the correct details to ensure they are recognized next time.

## 4. Viewing Past Conversations
Always click the **Load More** button to review the customer's history. This gives you vital context, such as past complaints or resolutions, before you start typing your first response.

### Pro-Tips for Amy:
- **Jump to Bottom:** Use this button to return to the latest message if you've been scrolling through history.
- **New Message Alert:** If a customer sends a message while you're reading history, a notification will pop up.
